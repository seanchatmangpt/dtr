# Generated DTR Documentation Tests

> Generated by `ggen sync` from `ontology/dtr-spec.ttl`.
> Re-run `ggen sync` any time you update the ontology.

## Quick Start

```bash
# 1. Add lib/pom-fragment.xml to your project pom.xml
# 2. Copy lib/tests/*.java to src/test/java/<package>/
# 3. Fill in the TODO placeholders with real content
# 4. Run tests to generate documentation
mvnd verify
# Output: target/docs/<ClassName>.md
```

## Pipeline

```
ontology/dtr-spec.ttl
    ↓  ggen sync
lib/tests/*.java          ← scaffolded @Test methods
    ↓  mvnd verify
target/docs/*.md          ← rendered documentation
```

## Modifying the Spec

Edit `ontology/dtr-spec.ttl`, then re-run `ggen sync`.

| Change | How |
|--------|-----|
| Add a section | Add a new `dtr:DocSection` and link it with `dtr:section` |
| Rename a section | Change `dtr:title` on the section resource |
| Add a say* method | Add `dtr:sayMethod "sayTable"` (or any other method) |
| Add a new class | Add a new `dtr:DocClass` with `dtr:package`, `dtr:name`, `dtr:section` |
| Change package | Edit `dtr:package` on the `dtr:DocClass` |
