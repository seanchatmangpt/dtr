# DTR CLI How-To Guides

This directory contains task-oriented guides to solve specific problems with the DTR CLI. Each guide walks you through a common workflow with step-by-step instructions and copy-paste ready examples.

## Guides Overview

### 1. [Export in Different Formats](formats.md) — 40 min read
**Problem:** I need to create PDF / slides / blog posts

Learn how to export Markdown to multiple output formats:
- Markdown (portable, Git-friendly)
- HTML (web viewing with styling)
- LaTeX/PDF (academic publishing with templates)
- HTML5 Slides (Reveal.js presentations)
- Blog posts (Dev.to, Medium, Hashnode)
- OpenAPI/Swagger specs

**Key commands:**
```bash
dtr export guide.md --format html
dtr export guide.md --format latex --template acm-conference
dtr export guide.md --format slides
```

### 3. [Integrate with Maven Builds](maven-integration.md) — 45 min read
**Problem:** I want docs auto-generated during my Maven build

Learn how to generate documentation automatically as part of your test suite:
- Add DTR plugin to pom.xml
- Configure output formats and directories
- Use Maven profiles for different outputs
- Control generation via CLI properties
- Integrate with GitHub Actions and Jenkins

**Key configuration:**
```xml
<plugin>
    <groupId>io.github.seanchatmangpt.dtr</groupId>
    <artifactId>dtr-maven-plugin</artifactId>
    <version>2.5.0</version>
    <configuration>
        <format>markdown</format>
        <outputDirectory>${project.basedir}/docs</outputDirectory>
    </configuration>
</plugin>
```

### 4. [Batch Export Multiple Files](batch-export.md) — 40 min read
**Problem:** I have 50+ markdown files and need to convert them all

Discover efficient batch processing techniques:
- Export entire directories with a single command
- Use glob patterns to match specific files
- Parallel processing for 3-5x speed improvement
- Error handling (skip failures or halt on error)
- Dry-run mode to preview conversions
- Custom output naming schemes

**Key commands:**
```bash
dtr export src/docs/ --format html --output-dir output/
dtr export "**/*.md" --format html --parallel 10
dtr export src/docs/ --format html --preserve-structure --dry-run
```

### 5. [Publish to Blogging Platforms](blog-publishing.md) — 60 min read
**Problem:** I want to share my docs on Dev.to, Medium, etc.

Learn how to cross-post documentation to multiple platforms:
- Supported platforms (Dev.to, Medium, Hashnode, LinkedIn, Substack)
- Setting up API authentication
- Adding metadata for platform-specific customization
- Publishing directly or using a review queue
- Scheduling posts for later
- Tracking cross-platform analytics

**Key commands:**
```bash
dtr publish guide.md --platform devto --live
dtr export guide.md --format blog --platform devto,medium --queue
dtr queue list && dtr queue publish --all
```

A comprehensive troubleshooting guide covering:

- LaTeX/PDF compilation issues

- Template and styling errors
