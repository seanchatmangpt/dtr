# DTR 2026.4.0 — DX/QoL Enhancement Release

## Release Information

- **Release Date:** 2026-03-16
- **Version:** 2026.4.0
- **Status:** DX/QoL Enhancement Release

## Overview

DTR 2026.4.0 is a **Developer Experience (DX) and Quality of Life (QoL) enhancement release** focused on improving the testing and documentation workflow. This release adds 12 new `say*` methods for better assertion integration, cross-references, and presentation support, while completing the migration to JUnit Jupiter 6.

**Key highlights:**
- 4 new `sayAndAssertThat` methods combine assertions with documentation in a single call
- 7 new presentation-specific methods for slides and blogs (saySlideOnly, sayTweetable, sayTldr, etc.)
- New `sayRef` overload for convenient cross-references
- JUnit Jupiter 5 → 6 migration complete
- Critical bug fix: DtrContext parameter injection now works correctly
- IDE integration support with LivePreview annotation

---

## What's New

### Added

#### Assertion + Documentation Combined

Four new `sayAndAssertThat` methods eliminate the boilerplate of asserting and documenting separately. Each method runs a Hamcrest assertion and automatically documents the result as "✓ PASS" on success.

```java
// Generic version
<T> void sayAndAssertThat(String label, T actual, Matcher<? super T> matcher)

// Primitive overloads (avoid autoboxing ambiguity)
void sayAndAssertThat(String label, long actual, Matcher<Long> matcher)
void sayAndAssertThat(String label, int actual, Matcher<Integer> matcher)
void sayAndAssertThat(String label, boolean actual, Matcher<Boolean> matcher)
```

**Example:**
```java
sayAndAssertThat("HTTP Status", response.getStatusCode(), equalTo(200));
sayAndAssertThat("Response Size", response.getBody().length(), greaterThan(0));
```

**Renders as:**
```
| Check | Result |
| --- | --- |
| HTTP Status | ✓ PASS |
| Response Size | ✓ PASS |
```

#### Cross-Reference Convenience Overload

New convenience overload for `sayRef` eliminates the need to manually construct `DocTestRef` objects.

```java
// Old way (still supported)
sayRef(DocTestRef.of(OtherDocTest.class, "user-registration"));

// New way (more concise)
sayRef(OtherDocTest.class, "user-registration");
```

#### Presentation-Specific Methods

Seven new methods enable content targeting for slides, blogs, and social media while keeping documentation tests DRY (Don't Repeat Yourself).

**Slide-Specific Content:**
```java
// Content appears ONLY in slide deck, not in generated docs
void saySlideOnly(String text)

// Presenter notes (slides only, visible to speaker during presentation)
void saySpeakerNote(String text)
```

**Documentation-Specific Content:**
```java
// Content appears ONLY in docs/blog, not in slides
void sayDocOnly(String text)
```

**Blog & Social Media:**
```java
// Hero image section (blogs + slides)
void sayHeroImage(String altText)

// Social-media quote box (≤280 chars for Twitter/X)
void sayTweetable(String text)

// TL;DR summary box (blogs)
void sayTldr(String text)

// Call-to-action button/link (blogs)
void sayCallToAction(String url)
```

**Example:**
```java
say("This is comprehensive documentation with all the details.");
sayDocOnly("See the API reference for complete parameter documentation.");

saySlideOnly("Key Point: Simplified for presentation");
saySpeakerNote("Mention that we're skipping the edge cases for brevity");

sayTweetable("DTR 2026.4.0 adds 12 new say* methods for better DX!");
sayTldr("New assertion methods + presentation support + JUnit Jupiter 6");
sayCallToAction("https://github.com/seanchatmangpt/dtr");
```

#### IDE Integration Support

New `@LivePreview` annotation enables IDE live preview functionality for documentation tests.

```java
@LivePreview
@Test
void myDocumentationTest() {
    say("This content can be previewed in supported IDEs");
}
```

IntelliJ IDEA configuration included in `META-INF/dtr/intellij.xml`.

### Changed

#### JUnit Jupiter Migration (Version 5 → 6)

Complete migration from JUnit Jupiter 5 to JUnit Jupiter 6.

**Dependency changes:**
- Updated: `org.junit.jupiter:junit-jupiter` from `5.x` to `6.0.3`
- Removed: `junit:junit:4.12` (legacy JUnit 4)
- All test code migrated to JUnit Jupiter 6 APIs

**Benefits:**
- Access to latest JUnit Jupiter 6 features
- Improved performance and test execution
- Better alignment with Java 26 ecosystem

#### Critical Bug Fix: DtrContext Parameter Injection

**Problem:** DtrExtension documented that `DtrContext` could be injected as a test parameter, but the implementation was missing the `ParameterResolver` interface.

**Error before fix:**
```
No ParameterResolver registered for parameter [io.github.seanchatmangpt.dtr.junit5.DtrContext arg0]
```

**Solution:**
- Added `ParameterResolver` to `DtrExtension` implements clause
- Implemented `supportsParameter()` to detect `DtrContext` type parameters
- Implemented `resolveParameter()` to inject `DtrContext` with shared `RenderMachine`
- Added `DtrExtensionParameterInjectionTest` to verify the fix

**Usage now works as documented:**
```java
@ExtendWith(DtrExtension.class)
class MyApiDocTest {

    @Test
    void testGetUsers(DtrContext ctx) {  // Parameter injection now works!
        ctx.sayNextSection("User API");
        ctx.say("Documentation for User API goes here.");
    }
}
```

## Breaking Changes

### Configuration Changes

**REMOVED:** `RenderConfig` class has been removed.

**REPLACEMENT:** Use `@DtrConfig` annotation or `DtrConfiguration` builder instead.

**Before (2026.3.0):**
```java
RenderConfig config = new RenderConfig();
List<RenderMachine> machines = config.createRenderMachines();
```

**After (2026.4.0):**
```java
@DtrConfig(format = OutputFormat.MARKDOWN)
class MyTest { }

// OR programmatic:
DtrConfiguration config = DtrConfiguration.builder()
    .format(OutputFormat.MARKDOWN)
    .build();
```

### Deprecated Methods

**DEPRECATED:** `sayCodeModel(Method method)` — Use `sayMethodSignature(Method method)` instead.

**Rationale:** The original `sayCodeModel(Method)` method name was ambiguous — it suggested full code model rendering (like `sayCodeModel(Class)`), but only rendered the method signature. The new name `sayMethodSignature` more accurately reflects the method's behavior.

**Timeline:** Removal planned for 2027.1.0

**Migration guide:**
```java
// Old (deprecated, still works)
sayCodeModel(myMethod);

// New (recommended)
sayMethodSignature(myMethod);
```

## Upgrade Guide

### From v2026.3.0 to v2026.4.0

**Step 1: Update dependency version**

```xml
<version>2026.4.0</version>
```

**Step 2: Update method calls**

If you're using `sayCodeModel(Method method)`, rename it to `sayMethodSignature(Method method)`:

```java
// Before
sayCodeModel(MyClass.class.getMethod("myMethod"));

// After
sayMethodSignature(MyClass.class.getMethod("myMethod"));
```

**Step 3: Update JUnit dependencies**

Ensure your `pom.xml` uses JUnit Jupiter 6:

```xml
<dependency>
  <groupId>org.junit.jupiter</groupId>
  <artifactId>junit-jupiter</artifactId>
  <version>6.0.3</version>
  <scope>test</scope>
</dependency>
```

**Step 4: Leverage new features (optional)**

Replace assertion + documentation boilerplate with `sayAndAssertThat`:

```java
// Before
assertThat(response.getStatus(), equalTo(200));
sayAssertions(Map.of("HTTP Status", "PASS"));

// After
sayAndAssertThat("HTTP Status", response.getStatus(), equalTo(200));
```

Add presentation-specific content for slides/blogs:

```java
say("Comprehensive documentation...");
sayTldr("Quick summary for busy readers");
saySlideOnly("Simplified bullet points for slides");
```
