# DTR 2026.1.0

**Released:** 2026-03-14

## Initial CalVer release

- Migrated from SemVer `2.6.0` to Calendar Versioning `YYYY.MINOR.PATCH`
- Version scheme: `2026.MINOR.PATCH` — year is the major, no `release-major` target
- Breaking changes handled by `@Deprecated` with minimum one-year removal window
- Year boundary (`2026.x.x` → `2027.1.0`) is a temporal transition, not a compatibility break
- `scripts/bump.sh` — CalVer arithmetic, year-reset on minor bump when calendar year advances
- `scripts/release.sh` — commit + tag + push, triggers GitHub Actions
- `scripts/release-rc.sh` — RC tags route to GitHub Packages only
- `scripts/changelog.sh` — generates `docs/releases/` and `docs/CHANGELOG.md` per release
- `Makefile` targets: `release-minor`, `release-patch`, `release-year`, `release-rc-minor`, `release-rc-patch`
- GitHub Actions: final tags → Maven Central; RC tags → GitHub Packages

## Install

```xml
<dependency>
  <groupId>io.github.seanchatmangpt.dtr</groupId>
  <artifactId>dtr-core</artifactId>
  <version>2026.1.0</version>
</dependency>
```

For year-bounded range (recommended for libraries):

```xml
<version>[2026.1.0,2027)</version>
```
