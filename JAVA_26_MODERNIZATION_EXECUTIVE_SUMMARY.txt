4. VIRTUAL THREADS (StructuredTaskScope) — Async I/O
   ───────────────────────────────────────────────
   Status: BenchmarkRunner already uses StructuredTaskScope ✓
   Priority: LOW-MEDIUM
   Effort: 2-3 hours
   Impact: 10-15% improvement on multi-document builds
   Files: RenderMachineLatex, DocMetadata, LaTeX compilation chain

   Opportunity:
     - Parallel LaTeX PDF compilation (currently sequential)
     - Parallel git/mvn metadata gathering (proposed in Opportunity 2)
     - Zero overhead vs. platform threads

   Example:
     BEFORE: for (Path texFile : texFiles) compiler.compile(texFile);  // Sequential
     AFTER:  try (var scope = StructuredTaskScope.ShutdownOnFailure<Void>()) {
               for (Path texFile : texFiles) {
                 scope.fork(() -> { compiler.compile(texFile); return null; });
               }
               scope.join();
             }  // Parallel

5. PATTERN MATCHING GUARDS (JEP 406)
   ────────────────────────────────
   Status: Switch statements ready for guards
   Priority: HIGH
   Effort: 1-2 hours
   Impact: 30% improvement in code clarity
   Files: RenderMachineImpl, RenderConfig, BibTeXRenderer, LatexCompiler

   Current: Simple switch statements on strings/integers
   Modernized: Pattern matching with guard conditions (when clause)

   Example:
     BEFORE: switch (clazz) {
               case Class<?> c when c.isRecord() -> "record";
               case Class<?> c when c.isInterface() -> "interface";
             }
     AFTER:  switch (clazz) {
               case Class<?> c when c.isRecord() && !c.isSealed() -> "record";
               case Class<?> c when c.isRecord() && c.isSealed() -> "sealed-record";
               case Class<?> c when c.isInterface() && c.isSealed() -> "sealed-interface";
               case Class<?> c when c.isInterface() -> "interface";
             }
