IMPLEMENTATION CORRECTNESS
================================================================================

Current Code (DocMetadata.java):

    private static final DocMetadata CACHED_INSTANCE = computeFromBuild();

    public static DocMetadata getInstance() {
        return CACHED_INSTANCE;
    }

How It Works:
  1. Class loads → JVM acquires implicit lock on class
  2. Static field initializer executes (computeFromBuild called once)
  3. External processes spawned once (git, mvn, hostname)
  4. Result stored in CACHED_INSTANCE
  5. Lock released
  6. Subsequent calls return pre-computed field (78 ns)

Why This Is Safe:
  - Java Language Spec guarantees thread-safe static initialization
  - No null checks needed (final field)
  - JIT compiler likely inlines to single field load instruction
  - Immutable after initialization (record + final)

Alternatives Considered:
  ✗ Double-Checked Locking: Extra null checks (slower)
  ✗ Lazy Initialization: Risk of race conditions
  ✗ Synchronization: Adds overhead to every access
  ✓ Eager Static Init: Fastest, simplest, safest
