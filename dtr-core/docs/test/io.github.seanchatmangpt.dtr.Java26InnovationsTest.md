## Innovation 1: Call Site Provenance via StackWalker

Every documentation section knows exactly where it was generated. No manual labeling. No string literals to maintain. The JVM stack carries provenance automatically.

`sayCallSite()` uses `StackWalker.getInstance(RETAIN_CLASS_REFERENCE)` to walk the live call stack, skip DTR's own frames, and surface the first frame from the actual test code.

```java
// StackWalker — Java 9+ (enhanced in Java 21+)
StackWalker.getInstance(StackWalker.Option.RETAIN_CLASS_REFERENCE)
    .walk(frames -> frames
        .filter(f -> !f.getClassName().startsWith("io.github.seanchatmangpt.dtr"))
        .findFirst())
    .ifPresent(frame -> {
        // frame.getClassName()  → test class
        // frame.getMethodName() → test method
        // frame.getLineNumber() → exact line
    });
```

## Innovation 3: Class Hierarchy as Auto-Generated Tree

Understanding a class's position in the type hierarchy is essential for API documentation. `sayClassHierarchy()` renders the complete superclass chain and all implemented interfaces as a visual tree.

No architecture diagram tool. No manual UML. The JVM reflection API carries the full hierarchy — `Class.getSuperclass()` walks the chain, `Class.getInterfaces()` lists the implemented interfaces.

```java
// Walk the superclass chain
Class<?> current = clazz;
var chain = new ArrayList<Class<?>>();
while (current != null) {
    chain.add(current);
    current = current.getSuperclass(); // null at Object's parent
}
Collections.reverse(chain); // Object → ... → clazz

// Render as indented tree
for (int i = 0; i < chain.size(); i++) {
    String indent = "  ".repeat(i);
    markdownDocument.add(indent + (i == 0 ? "" : "↳ ") + className);
}
```

## Innovation 4: String Profile via Java String APIs

When testing text processing APIs, JSON serializers, or template engines, the structure of string payloads matters. `sayStringProfile()` renders word count, line count, character category distribution, and Unicode metrics for any string — using only `String.chars()`, `String.lines()`, and `Character` utility methods.

```java
// Pure Java string APIs — no regex library, no external deps
long words = Arrays.stream(text.trim().split("\\s+"))
    .filter(w -> !w.isEmpty()).count();
long lines     = text.lines().count();
long unique    = text.chars().distinct().count();
long letters   = text.chars().filter(Character::isLetter).count();
long nonAscii  = text.chars().filter(c -> c > 127).count();
```

## Innovation 5: Reflective Diff — What Changed, Automatically

API documentation should show not just the current state but how things evolve. `sayReflectiveDiff()` uses `Class.getDeclaredFields()` with `setAccessible(true)` to compare two objects field-by-field and render a diff table.

No equals() override needed. No custom comparison logic. The reflection API sees every declared field.

```java
// Reflective field-by-field comparison
for (var field : before.getClass().getDeclaredFields()) {
    field.setAccessible(true); // bypasses access control
    Object beforeVal = field.get(before);
    Object afterVal  = field.get(after);
    boolean changed  = !Objects.equals(beforeVal, afterVal);
    // render the row
}
```
